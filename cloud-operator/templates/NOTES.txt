### VPC CNI Flow Collection

{{- if not (.Capabilities.APIVersions.Has "networking.k8s.aws/v1alpha1") }}

WARNING: VPC CNI flow collection is enabled but ClusterNetworkPolicy CRD was not found.

The CRD (networking.k8s.aws/v1alpha1) is required for comprehensive flow logging.
Without it, flow logs will only be emitted for pods with explicit NetworkPolicy.

To fix this:
1. Ensure you're running on EKS (not Auto Mode or Fargate)
2. Enable Network Policy in VPC CNI addon:

   aws eks update-addon --cluster-name <CLUSTER> --addon-name vpc-cni \
     --configuration-values '{"enableNetworkPolicy":"true","nodeAgent":{"enablePolicyEventLogs":"true"}}' \
     --resolve-conflicts OVERWRITE

3. Run helm upgrade to create the ClusterNetworkPolicy:

   helm upgrade {{ .Release.Name }} <chart> --namespace {{ .Release.Namespace }}
